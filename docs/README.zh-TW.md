# SSE 即時系統監控平台
### （本文使用豆包AI翻譯，AI可能會犯錯，望周知）
基於 **Node.js + Express + SSE** 開發的輕量級 Linux 伺服器即時監控系統，透過 Server-Sent Events（SSE）技術實現毫秒級資料推送，不需前端輪詢，低資源耗損、高即時性。

## ✨ 專案特色
- **即時資料推送**：基於 SSE 長連線，500ms 等級系統資料即時推送
- **完整監控指標**：CPU、記憶體、Swap、磁碟、網路、系統資訊全方位涵蓋
- **低資源耗損**：純原生 Node.js 實作，無複雜相依套件，伺服器端資源消耗極低
- **開箱即用**：一行指令啟動，不需複雜設定
- **視覺化介面**：簡潔美觀的 Web 前端，即時查看所有監控資料
- **穩定可靠**：完整的例外捕捉、連線中斷自動重新連線機制

## 📊 監控指標一覽
| 模組 | 監控內容 |
|------|----------|
| **CPU** | 使用率、核心數、1/5/15分鐘系統負載 |
| **記憶體** | 總容量、已用/閒置、使用率百分比 |
| **Swap** | Swap 分割區使用狀況、使用率 |
| **磁碟** | 掛載點、使用率 |
| **網路** | 即時上下傳速度、累計總流量、網卡資訊 |
| **系統** | 主機名稱、系統版本、核心版本、運行時間、行程數 |
| **網路** | SSE 連線狀態 |

## 🕒 額外特色功能
- **雙時鐘顯示**：透過 SSE 即時推送後端時間，前端精確顯示伺服端時間與本地時間
- **網路延遲即時偵測**：自動 ping 測試，即時顯示目前伺服器連線延遲

## 🔧 核心技術說明
### 1. Server-Sent Events (SSE)
- 伺服器主動向客戶端推送即時資料的 HTML5 技術
- 相較於 WebSocket 更輕量，僅支援伺服器→客戶端單向推送
- 原生支援斷線重連，非常適合監控場景

### 2. 系統資訊擷取原理
- 讀取 Linux 系統 `/proc` 目錄下的系統檔案取得即時資料
- CPU/記憶體/網路/行程資料皆從核心原生介面取得，精準無誤差
- 磁碟資訊透過 `df -h` 指令解析，相容主流 Linux 發行版

## 📝 API 介面說明
| 介面 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 回傳前端監控頁面 |
| `/api/ping` | POST | 延遲測試介面，計算網路耗時 |
| `/sse/time` | GET | SSE 即時時間推送 |
| `/sse/system` | GET | SSE 系統監控資料推送 |

## 🛡️ 安全性與穩定性
- 所有系統指令執行皆加入例外捕捉，服務不會當機
- 客戶端中斷連線時自動清除計時器，避免記憶體洩漏
- 前端 SSE 斷線自動重連，確保監控不中斷
- 僅讀取系統資訊，無任何寫入操作，安全無風險
