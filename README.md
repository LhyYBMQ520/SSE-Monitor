# SSE 实时系统监控平台
基于 **Node.js + Express + SSE** 开发的轻量级 Linux 服务器实时监控系统，通过 Server-Sent Events 技术实现毫秒级系统数据推送，无需前端轮询，低资源占用、高实时性。

## ✨ 项目特性
- **实时数据推送**：基于 SSE 长连接，500ms 级系统数据实时推送
- **全面监控指标**：CPU、内存、Swap、磁盘、网络、系统信息全维度覆盖
- **低资源占用**：纯原生 Node.js 实现，无 heavy 依赖，服务端资源消耗极低
- **开箱即用**：一行命令启动，无需复杂配置
- **可视化界面**：简洁美观的 Web 前端，实时查看所有监控数据
- **稳定可靠**：完善的异常捕获、连接断开自动重连机制

## 📊 监控指标一览
| 模块 | 监控内容 |
|------|----------|
| **CPU** | 使用率、核心数、1/5/15分钟系统负载 |
| **内存** | 总容量、已用/空闲、使用率百分比 |
| **Swap** | Swap 分区使用情况、使用率 |
| **磁盘** | 各分区容量、已用、挂载点、使用率 |
| **网络** | 实时上下行速度、累计总流量、网卡信息 |
| **系统** | 主机名、系统版本、内核版本、运行时长、进程数 |
| **网络** | 服务器连接延迟、SSE 连接状态 |

## 🚀 快速开始
### 1. 环境要求
- Linux 系统（项目依赖 `/proc` 系统文件，仅支持 Linux）
- Node.js 14+

### 2. 安装运行
```bash
# 1. 克隆/下载项目文件
# 2. 安装依赖
npm install express

# 3. 启动服务
node server.js
```

### 3. 访问使用
启动成功后浏览器访问：
```
http://服务器IP:23456
```
即可查看实时监控面板。

## 🔧 核心技术说明
### 1. Server-Sent Events (SSE)
- 服务端主动向客户端推送实时数据的 HTML5 技术
- 相比 WebSocket 更轻量，仅支持服务端→客户端单向推送
- 原生支持断线重连，适配监控场景

### 2. 系统信息采集原理
- 读取 Linux 系统 `/proc` 目录下的系统文件获取实时数据
- CPU/内存/网络/进程数据均从内核原生接口获取，精准无误差
- 磁盘信息通过 `df -h` 命令解析，兼容主流 Linux 发行版

## ⚙️ 自定义配置
### 修改服务端口
打开 `server.js`，修改 `PORT` 常量：
```javascript
const PORT = 23456; // 改为你需要的端口
```

### 修改数据推送频率
打开 `server.js`，修改定时器间隔：
```javascript
// 时间推送（默认 500ms）
setInterval(() => {}, 500);
// 系统监控推送（默认 500ms）
setInterval(async () => {}, 500);
```

## 📝 API 接口说明
| 接口 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 返回前端监控页面 |
| `/api/ping` | POST | 延迟测试接口，计算网络耗时 |
| `/sse/time` | GET | SSE 实时时间推送 |
| `/sse/system` | GET | SSE 系统监控数据推送 |

## 🛡️ 安全与稳定性
- 所有系统命令执行均添加异常捕获，服务不会崩溃
- 客户端断开连接自动清理定时器，避免内存泄漏
- 前端 SSE 断开自动重连，保证监控连续性
- 仅读取系统信息，无任何写入操作，安全无风险
